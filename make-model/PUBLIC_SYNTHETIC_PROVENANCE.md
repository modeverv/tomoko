# Public Synthetic Saturation Model Provenance

This document describes the intended public-friendly training path for Tomoko's
tiny semantic saturation scorer.

## Data Sources

- Input utterance text is generated by `generate_synthetic_saturation_corpus.py`.
- The generator uses repository-owned templates authored through Codex/user
  collaboration.
- No internet dialogue corpus is used for this public-synthetic line.
- Japanese Daily Dialogue, BSD corpus, Common Voice, and other third-party
  corpora are not inputs to this line.

## Teacher Labels

- Initial `SATURATION=0.0..1.0` labels are generated by Gemma 4 26B.
- Gemma 4 is used as a teacher model for this narrow regression target.
- Additional anchor labels are generated from repository-owned handcrafted
  templates and Codex/user adjustments.

## Student Model

- The student model is a hash-ridge saturation scorer.
- The artifact stores numeric weights, a bias, config, and metadata.
- It does not store a vocabulary or source utterance dictionary.
- It predicts conversational semantic saturation only; it is not a general
  language model.

## Intended Public Artifact

The intended public artifact is:

```text
make-model/artifacts/public-synthetic-gemma26b-plus-anchors-life-h8192-l001-saturation-model.json
```

The corresponding training data can be regenerated from scripts in this
directory. Generated datasets under `make-model/data/` and trained artifacts
under `make-model/artifacts/` may be kept out of git until explicitly approved
for publication.
